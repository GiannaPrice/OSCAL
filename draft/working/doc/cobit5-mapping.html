IT-related goal table enhancement with @class='goal', containing 'scope' prop = 'IT'

Each IT goal presents an implicit link to an IT goal specified elsewhere in COBITS (the Goals Cascade). Here, references to IT goals are indicated by a two-digit numeric identifier (01-17) and a title, and glossed with a brief statement, Metrics. These are captured inside the enhancement as prop with @class='number', title and stmt with @class='metrics'.

The fact that this is an IT-related control enhancement is captured by including a property prop with @class='scope' with the value IT.

The control enhancement used to describe this is the same as is used for the next table, the process-related goal table. Note however that the IT-related goals (as indicated by the scope property) are implicit cross-links to elsewhere in OSCAL.

Process Goal and Metrics (table) enhancement with @class='goal', containing 'scope' prop ='process'

Like IT-related goals, each process-related goal has a number, title and Metrics section. In this case, however, the property prop with @class='scope' has the value process.

RACI chart enhancement with @class='raci_index' inside enhancement with @class='process'

Presents a map of responsibilities, role category classifications for responsible, accountable, consulted, informed.

Since the table allocates the values across processes (control enhancements), its contents are allocated to those elements (i.e. disbursed) in the OSCAL.

Practice inputs enhancement with @class='input'

Note that according to COBIT (p. 26) not all practice inputs are represented, but only those that are specific to particular practices. So if a practice has an input due to another practice having all or all EDM as an output (for example), such an input is not listed. Only inputs designated for this particular control (practice) are indicated.

Implicit inputs can be detected by querying.
